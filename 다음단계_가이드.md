## 📋 다음 단계 체크리스트

### 1단계: POSTGRES_PRISMA_URL 확인

**확인 방법:**
1. Vercel 대시보드 → Settings → Environment Variables
2. `POSTGRES_PRISMA_URL`이 목록에 있는지 확인

**상황별 대응:**

#### ✅ POSTGRES_PRISMA_URL이 있으면
→ 다음 단계로 진행하세요!

#### ❌ POSTGRES_PRISMA_URL이 없으면
→ Postgres 데이터베이스를 생성해야 합니다:

1. Vercel 대시보드 상단 메뉴에서 **Storage** 탭 클릭
2. **Prisma Postgres** 또는 **Neon** 찾기
3. **Create** 버튼 클릭
4. 데이터베이스 이름 입력 (예: `newschatbot-db`)
5. **Create** 클릭
6. 생성 완료 후 `POSTGRES_PRISMA_URL`이 자동으로 추가됨

---

### 2단계: GitHub에 코드 푸시 확인

**확인 사항:**
- GitHub 저장소에 최신 코드가 업로드되었는지 확인
- https://github.com/jwkyung/NewsChatBot 접속하여 확인

**아직 업로드하지 않았다면:**

#### 방법 1: GitHub Desktop 사용
1. GitHub Desktop 실행
2. File → Add Local Repository
3. `C:\Users\SD2-20\Desktop\chatbot\github-upload` 폴더 선택
4. Summary 입력: `데이터베이스 기능 추가`
5. **Commit to main** 클릭
6. **Push origin** 클릭

#### 방법 2: GitHub 웹사이트
1. https://github.com/jwkyung/NewsChatBot 접속
2. **Add file → Upload files** 클릭
3. `github-upload` 폴더의 모든 파일 업로드
4. **Commit changes** 클릭

---

### 3단계: Vercel 자동 배포 확인

**확인 방법:**
1. Vercel 대시보드 → **Deployments** 탭
2. 최신 배포 상태 확인:
   - ✅ **Ready** (초록색) = 배포 성공
   - ⏳ **Building** = 배포 중
   - ❌ **Error** = 오류 발생 (로그 확인 필요)

**배포가 자동으로 시작되지 않으면:**
- GitHub에 푸시했는지 확인
- Vercel과 GitHub 저장소가 연결되어 있는지 확인

---

### 4단계: 배포 완료 후 테스트

**배포가 성공하면:**
1. Vercel 대시보드에서 배포된 URL 확인
2. 브라우저에서 사이트 접속
3. 기능 테스트:
   - ✅ 뉴스 검색 테스트
   - ✅ 검색 기록 버튼 클릭 테스트
   - ✅ 챗봇 기능 테스트

## 🔧 문제 해결

### 배포 실패 시

**확인 사항:**
1. **Deployments** → 최신 배포 클릭 → **Logs** 확인
2. 오류 메시지 확인:
   - "Prisma Client has not been generated" → `package.json`의 `postinstall` 확인
   - "Database does not exist" → `POSTGRES_PRISMA_URL` 환경변수 확인
   - "GEMINI_API_KEY not found" → 환경변수 다시 확인

### 환경변수 문제

**재배포 필요:**
- 환경변수를 추가/수정한 후에는 수동으로 재배포해야 할 수 있습니다
- **Deployments** → 최신 배포 → **Redeploy** 클릭
